## Лабораторна 1: Custom Memory Allocator (Best Fit & Red-Black Tree)

Цей проєкт — це власна реалізація системного алокатора пам'яті на мові C, що імітує роботу стандартних функцій `malloc`, `free` та `realloc`. Алокатор оптимізований для боротьби з фрагментацією пам'яті, підтримує роботу з кількома аренами та використовує Червоно-чорне дерево для швидкого пошуку вільних блоків за стратегією **Best Fit** (O(log N)).

Проєкт розроблено з підтримкою ОС Windows (WinAPI) та можливістю роботи зі сторінковою пам'яттю (Memory Discarding & Poisoning).

Файли можна переглянти в папці [lab1](https://github.com/SonikLvl/SPZ_sem6/tree/main/lab1).

### Архітектура проєкту та опис файлів

### Конфігурація та утиліти
* **`config.h`**
    Містить базові налаштування системи (розмір сторінки `PAGE_SIZE`, початковий розмір арени `DEFAULT_ARENA_SIZE`) та перемикач `USE_PAGED_MEMORY` для підтримки несторінкової пам'яті.
* **`allocator_impl.h`**
    Внутрішні макроси алокатора, зокрема константи вирівнювання (`ALIGNMENT`) та математика округлення розмірів.

### Взаємодія з ядром ОС (OS Abstraction Layer)
* **`kernel.h`** / **`kernel.c`**
    Ізольований шар для системних викликів. Реалізує отримання та повернення сирої пам'яті в ядро Windows за допомогою `VirtualAlloc` / `VirtualFree`. Також містить функцію `os_discard_pages` для інформування ОС про порожні сторінки всередині вільних блоків (з функціоналом Memory Poisoning для автотестування).

### Структура блоку пам'яті
* **`block.h`**
    Описує структуру заголовка блоку (`block_header_t`). Використовує побітові маски для зберігання прапорців (`busy`, `first`, `last`) у молодших бітах розміру блоку. Надає inline-функції для навігації пам'яттю.
* **`block.c`**
    Реалізує низькорівневі операції над фізичними блоками: `block_split` (поділ великого блоку на два менших) та `block_merge` (злиття суміжних вільних блоків за O(1)).

### Структури даних (Best Fit Search)
* **`rbtree.h`** / **`rbtree.c`**
    Класична реалізація Червоно-чорного дерева. Використовується для організації всіх вільних блоків. Вершини дерева зберігаються безпосередньо у вільному просторі (payload) самих блоків, що економить пам'ять.
* **`tree.h`**
    Файл-адаптер. Створює синоніми та макроси, що дозволяють відв'язати логіку алокатора від конкретної реалізації дерева. Дозволяє легко замінити R-B Tree на AVL-дерево в майбутньому.

### Головне API Алокатора
* **`allocator.h`**
    Публічний інтерфейс бібліотеки (`mem_alloc`, `mem_free`, `mem_realloc`, `mem_show`).
* **`allocator.c`**
    Високорівнева логіка управління пам'яттю:
    * Управління множинними аренами (зокрема величезними блоками понад дефолтний розмір арени).
    * Делегування пошуку Best Fit до структури дерева.
    * Оптимізація `mem_realloc` для in-place розширення/зменшення блоків (з урахуванням `SHRINK_THRESHOLD`).

### Тестування
* **`main.c`**
    Точка входу, яка містить агресивний **Автоматичний Fuzzing-тестер**. Виконує 10 000 випадкових операцій алокації, реаллокації та звільнення пам'яті. Тестер заповнює блоки випадковими даними та перевіряє їхню цілісність за допомогою контрольних сум (Checksums), гарантуючи відсутність перекриття блоків (memory corruption).
